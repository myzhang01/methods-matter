A guide to installing the necessary packages to run the analysis files for users who access Stata through Citrix can be found below:

0. Change your working directory using the `cd` command and appending `\\Client` to your file path
    - For example, if my reproduction files are located in `C:\Users\myzha\Documents\GitHub\reprod-brodeur-cook-heyes`, every time I start up Stata I would enter in the command `cd \\Client\C:\Users\myzha\Documents\GitHub\reprod-brodeur-cook-heyes`

1. 3 packages, `yaml`, `matrix_to_txt`, and `preliminaries` must be downloaded manually from GitHub and then installed into your system directory for user-written packages (`PLUS`)
    - Somewhere on your local machine, create a folder to store the user-written packages. My folder is located at the path `C:\Users\myzha\Documents\Stata\ado`
      - Every time you start up Stata, enter the command `sysdir set PLUS \\Client\insert_your_file_path_here`. I would enter `sysdir set PLUS \\Client\C:\Users\myzha\Documents\Stata\ado`
    - Go to [this GitHub repo](https://github.com/gslab-econ/stata-misc) to download the `yaml` package
      - Download a ZIP of the files and extract them. Make sure `stata.toc`, `yaml.ado`, `yaml.pkg`, `yaml.sthlp` are present
      - Enter `net install yaml, force from (\\Client\insert_your_file_path_here)`, with your file path being your download location. For example, I would enter `net install yaml, force from (\\Client\C:\Users\myzha\Downloads\stata-misc-master)`
      - Enter `which yaml` into Stata to confirm the package was successfully installed
    - Go to [another GitHub repo](https://github.com/gslab-econ/gslab_stata) to download the `matrix_to_txt` and `preliminaries` packages
      -  Download a ZIP of the files and extract them. Make sure `stata.toc`, `matrix_to_txt.ado`, `matrix_to_txt.pkg`, `matrix_to_txt.hlp`, `preliminaries.ado`, `preliminaries.pkg`, and `preliminaries.hlp` are present
      - Enter `net install matrix_to_txt, force from (\\Client\insert_your_file_path_here)`, with your file path being your download location. For example, I would enter `net install matrix_to_txt, force from (\\Client\C:\Users\myzha\Downloads\gslab_stata-master\gslab_misc\ado)`
      - Enter `net install preliminaries, force from (\\Client\insert_your_file_path_here)`, with your file path being your download location
      - Enter `which matrix_to_txt` and `which preliminaries` into Stata to confirm the package was successfully installed

2. 2 more packages, `dataout` and `distinct`, must be installed
    - Enter `ssc install dataout`
    - Enter `ssc install distinct`

3. Open `setup.do` to install the final 2 packages, `egenmore` and `moremata`
    - Select lines 5 to 18 and execute them to install `egenmore`
    - Select lines 27 to 32 and execute them to install `moremata`

You should now be ready to run all the `.do` files with no issues!

Thank you to the Urban Institute, whose readme at [this GitHub repo](https://github.com/UrbanInstitute/education-data-package-stata) helped me devise this package installation solution.
